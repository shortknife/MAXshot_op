## 1. Decision

Step 2 is officially reclassified from:
- previously: "frozen contract + partial implementation"
- now: "transitional shell that must be redone"

This is not a minor patch task.
This is a standards reset under v5.2.

## 2. Why Step 2 must be redone

The old Step 2 was designed as a conversation context manager, but it does not satisfy the current v5.2 standard in four decisive ways:

1. It is still too rule-heavy.
2. LLM is not yet the formal primary judge of turn relation.
3. `ContextPacket` is not yet the one stable runtime artifact.
4. TG / Web multi-turn behavior has not yet been hardened under one acceptance baseline.

## 3. What remains valid from the old Step 2

These parts are still valid and should be retained conceptually:

1. Step 2 belongs strictly before intent/capability matching.
2. Step 2 is responsible for session/thread relation, not capability choice.
3. Step 2 should output one standard context artifact.
4. Step 2 should preserve a deterministic policy boundary.
5. Session TTL / clarification store / memory runtime shell remain useful building blocks.

## 4. What is no longer accepted as the official target

The following are no longer acceptable as Step 2 target behavior:

1. Heuristic-heavy relation logic as the main path.
2. Query-string patching as the primary follow-up mechanism.
3. Implicit context mutation across helpers.
4. Business-only conversation context model.
5. “Works in some flows” without TG/Web unified acceptance.

## 5. Current implementation classification

| Area | Current state | v5.2 status |
|---|---|---|
| Session shell | Exists | Keep |
| Clarification state | Exists | Keep and formalize |
| Follow-up handling | Exists | Redesign |
| Relation classification | Partial heuristics | Replace as main path |
| Context artifact | Partial / implicit | Redesign as strict output |
| Memory runtime shell | Exists | Keep |
| Entry consistency across TG/Web | Partial | Formal acceptance missing |

## 6. Official completion criteria for Step 2

Step 2 is only considered done when all conditions below are true:

1. LLM is the primary judge of turn relation.
2. Code only enforces TTL / inheritance / policy boundaries.
3. Step 2 emits a single strict `ContextPacket` artifact.
4. Turn relation taxonomy is fixed and runtime-enforced.
5. TG / Web multi-turn behavior passes one shared acceptance matrix.
6. Old heuristic continuation logic is demoted to fallback only.

## 7. Immediate implication

No Step 2 coding should proceed under the old frozen V1 checklist.

From 2026-03-26 onward:
- old Step 2 docs are historical references
- new Step 2 V2 docs are the only valid implementation target
