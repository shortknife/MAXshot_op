# Step 3 测试矩阵 V1

> 范围：只验证 Intent Harness  
> 不验证：Query Contract 执行结果、SQL 正确性、业务数据真实性

## 1. 测试目标

验证 Step 3 是否能在 Step 2 冻结基线上，稳定输出：

1. 正确 capability match
2. 正确 slots
3. 正确 clarification decision
4. 正确 out_of_scope / capability_overview 识别

## 2. 核心检查字段

每条测试至少检查：

1. `matched_capability_ids`
2. `matched_capability_id`
3. `intent_type`
4. `need_clarification`
5. `slots.scope`
6. `slots.metric`
7. `slots.aggregation`
8. 时间相关字段：
   - `date_from/date_to`
   - `calendar_year`
   - `calendar_month`
   - `week_of_month`
   - `exact_day`

## 3. 基线测试组

### Group A - explicit calendar semantics

1. `3月第一周的平均APY是多少`
   - capability: `capability.data_fact_query`
   - scope: `yield`
   - metric: `apy`
   - aggregation: `avg`
   - month: `3`
   - week_of_month: `1`
   - no clarification

2. `2月份平均日tvl是多少`
   - capability: `capability.data_fact_query`
   - scope: `vault`
   - metric: `tvl`
   - aggregation: `avg`
   - month: `2`
   - no clarification

3. `3月20日当天APY最高和最低的vault是那两个呢？`
   - capability: `capability.data_fact_query`
   - scope: `yield`
   - metric: `apy`
   - question_shape: `top_bottom_in_day`
   - exact_day preserved
   - no clarification

### Group B - relation-aware follow-up consumption

Assume Step 2 already provided relevant context.

4. `那其TVL多少呢？`
   - capability: `capability.data_fact_query`
   - metric: `tvl`
   - inherits target object
   - inherits exact day if present
   - no time clarification

5. `当天有调仓的Action么？`
   - capability: `capability.data_fact_query`
   - scope: `rebalance`
   - metric: `rebalance_action`
   - exact day inherited
   - no generic APY fallback

6. `Base呢`
   - when prior question is vault list on ARB
   - capability: `capability.data_fact_query`
   - scope: `vault`
   - metric: `vault_list`
   - chain switches to `base`

7. `Sol呢`
   - same expectations
   - chain switches to `solana`

### Group C - capability explanation / non-metrics

8. `你能做什么业务呢？`
   - capability: `capability.product_doc_qna`
   - intent_type: `general_qna`
   - `question_shape=capability_overview`
   - must not match business metric capability

9. `MAXshot 产品逻辑是什么？`
   - capability: `capability.product_doc_qna`
   - no silent business metric routing

### Group D - reject / unknown

10. `帮我转 100 USDC`
   - `in_scope=false`
   - no executable capability

11. `今天天气怎么样`
   - `in_scope=false` or friendly non-business handling
   - no business capability match

## 4. Fail conditions

任一条出现以下情况，判定 Step 3 失败：

1. 显式时间语义被改写成 `最近7天`
2. `TVL` 被改写成 `APY`
3. `vault_list` 被改写成 `yield`
4. capability-overview 被改写成数据查询
5. follow-up 丢失上一轮 target object/day
6. clarification 本不该出现却出现
7. capability 匹配超过 3 个或空匹配仍被当成 in_scope 成功

## 5. 验收门槛

1. Group A 全通过
2. Group B 至少 3/4 通过
3. Group C 全通过
4. Group D 全通过
5. 之后 `phase0` 不回归
