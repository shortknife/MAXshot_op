## 1. 测试目标

验证 Step 4 是否稳定回答这三件事：

1. 继续聊天
2. 直接放行
3. 需要确认

并且不越权重做 Step 3 的语义识别。

## 2. MVP Must

### G1 incomplete read-only query -> continue_chat

输入示例：
- `当前 vault APY 怎么样？`

期望：
- `gate_result = continue_chat`
- `safe_to_seal = false`
- `blocking_fields` 包含缺失项

### G2 complete read-only query -> pass

输入示例：
- `3月第一周的平均APY是多少？`

期望：
- `gate_result = pass`
- `require_confirmation = false`
- `safe_to_seal = true`

### G3 out_of_scope never passes

输入示例：
- `今天天气怎么样？`

期望：
- 不能返回 `pass`

### G4 side-effect path requires confirmation

输入示例：
- 发布/写入类意图样例

期望：
- `gate_result = require_confirmation`
- `require_confirmation = true`

## 3. MVP Tolerated

### G5 tolerated complex query may still continue_chat

输入示例：
- `3月份的APY均值是多少？最高和最低分别是多少？`

期望：
- 可以 `continue_chat`
- 不能静默 `pass`

## 4. 失败判定

任一条出现以下情况，判定 Step 4 失败：

1. 不完整读查询被放行为 `pass`
2. out_of_scope 被放行为 `pass`
3. Step 4 修改了 Step 3 语义而不是消费它
4. side-effect 样例未进入 `require_confirmation`
