# Step 8 测试矩阵 V1

- Date: 2026-03-29
- Scope: `Step 8 Trace + Audit`
- Status: Active

## A. Audit write path

1. create path seeds `execution_created`
2. blocked run path appends `router_blocked`
3. successful run appends `router_start`, `capability_executed`, `router_complete`
4. replay appends `execution_replay_requested`
5. retry appends retry marker without dropping prior events

## B. Audit read path

1. canonical audit read returns normalized ordered events
2. missing optional fields are backfilled safely
3. older audit payload variants remain readable

## C. Integrity rules

1. every event contains `execution_id`
2. every event contains `timestamp`
3. event order is stable
4. write path is append-only

## D. Downstream compatibility

1. `/audit` read path consumes normalized events
2. `/lineage` replay nodes still derive from normalized audit events
3. Step 9 downstream can consume one stable trace read model
