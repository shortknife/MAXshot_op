# Step 6 测试矩阵 V1

- Date: 2026-03-29
- Scope: `Step 6 Router`
- Status: Active

## A. Runnable gating

1. non-runnable execution status -> router rejects dispatch
2. missing execution record -> router fails explicitly
3. runnable execution -> router starts and updates runtime status

## B. Capability authority

1. sealed primary capability id -> router binds the same primary capability
2. invalid primary capability id -> router rejects
3. router does not reread raw query to pick a different capability

## C. Decomposition and scope stability

1. yield execution -> capability chain stays yield-oriented
2. execution lookup -> not polluted by prior vault/yield context
3. compatibility-heavy sealed payload -> normalizes to canonical capability chain

## D. Audit and runtime outputs

1. audit contains `router_start`
2. audit contains `task_decomposed`
3. audit contains `memory_selected`
4. audit contains `capability_executed`
5. audit ends in `router_complete` or `router_error`
