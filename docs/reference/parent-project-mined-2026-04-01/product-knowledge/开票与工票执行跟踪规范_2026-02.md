# Decision_Layer 开票与工票执行跟踪规范

> **版本**：v1.0  
> **创建日期**：2026-02  
> **维护者**：Decision_Layer (LEO)  
> **用途**：统一开票流程、减少 TODO 更新频率、集中协作反馈

---

## 一、核心规则

**开票时**：
1. 创建 **本批工票执行跟踪文档**（格式见模板）
2. 在 **各被分配人 TODO** 中写入 **一条协作条**，指向该文档
3. **后续所有进展/阻塞/回复** 均在该执行跟踪文档中进行，不再频繁更新各角色 TODO

**被分配人**（Alex/Sam/Lily 等）：
- 收到 TODO 协作条后，打开执行跟踪文档查看工票
- 进展、阻塞、问题、交付物路径 → **直接写在执行跟踪文档** 对应工票行或协作留言区
- 不再在各角色 TODO 中重复填写进展

---

## 二、执行跟踪文档

### 2.1 命名与位置

```
Decision_Layer/4.Working/批次_{批次名}_工票执行跟踪_{YYYY-MM}.md
```

**示例**：`批次_SQL_Engine_Tier1_Tier3_工票执行跟踪_2026-02.md`

### 2.2 结构（必含）

| 区块 | 内容 |
|------|------|
| **一、批次概要** | 批次名、工票包路径、开票日期、当前状态 |
| **二、工票状态表** | 工票 | 负责人 | 状态 | 进展/交付物/阻塞 |
| **三、协作留言区** | 各方进展更新、阻塞说明、Decision_Layer 回复 |

### 2.3 状态取值

- `⏳ 待开始`
- `🔄 进行中`
- `✅ 完成`
- `⚠️ 阻塞`

---

## 三、TODO 协作条格式

在各角色 TODO 中写入一条，格式：

```
[Decision_Layer→角色] P? 本批工票见 {执行跟踪文档路径}；进展/反馈请直接在该文档填写。
```

**示例**：
```
[Decision_Layer→Alex] P1 本批工票见 Decision_Layer/4.Working/批次_SQL_Engine_Tier1_Tier3_工票执行跟踪_2026-02.md；进展/阻塞/交付物请直接在该文档 §二§三 填写。
```

---

## 四、执行流程

```
开票 → 创建执行跟踪文档（按模板） → 各角色 TODO 写一条引用 → 被分配人在执行跟踪文档反馈 → Decision_Layer 在该文档查看进展并回复
```
