# 04.5 Evolution Engine 与 Insight 形成 ⭐

> **用途**：Evolution 是 Working Mind 的时间维延展，不是独立服务  
> **v5.1 收敛**：不做复杂 Self-Reflection，先做「策略闭环」；进化只能发生在策略层，不得污染执行层。

---

## 一、定位

Evolution Engine 回答的不是「我现在怎么想」，而是 **「我为什么会慢慢变成现在这样」**。

---

## 二、闭环流程（v5.1 分阶段）

**v5.1 当前执行**（简化版，不做复杂 Self-Reflection）：

| 阶段 | 目标 | 说明 |
|------|------|------|
| **Phase 1（P1）** | 策略闭环基础 | 每篇内容打标签、记录策略参数、记录发布结果 |
| **Phase 2（P2）** | 简单归因 | 简单多因子归因、周期复盘报告 |
| **Phase 3（P3）** | 引入 Insight | Insight Memory、Hypothesis、推荐式优化 |

**v5.0 愿景**（完整闭环，P3 后启用）：
1. Goal Setting → 2. Hypothesis → 3. Action → 4. Attribution → 5. Self-Reflection → Insight Memory

---

## 三、Insight 来源

**Insight 只能从 Execution → Attribution → Reflection 里长出来。**

- 不是凭空生成
- 不是 LLM 自由发挥
- 必须关联 source_execution_id，可溯源
- **v5.1**：Phase 3 前不沉淀 Insight，仅做标签与归因记录

---

## 四、Evolution 与 Router 的关系

- **Evolution 不直接控制执行**
- **Evolution 只影响下一次 Working Mind 的构成**
- 对 Router 的影响只能通过 **Recommendation（可审计）**

---

## 五、升格时机

当 Insight 有独立存储、Attribution 有复杂 pipeline、Evolution 需要调度窗口时，再考虑升格为独立模块（11_Evolution_Engine）。  
当前阶段：Evolution 是 04 的一部分。

---

## 六、产品决议（本版唯一执行）

**决议 7（Evolution 仅通过 Recommendation 影响 Router）**：§四 已体现；Evolution 不自动改逻辑，对 Router 的影响只能通过 Recommendation（可审计）。开发/验收以 03.3、02.1 为准。
