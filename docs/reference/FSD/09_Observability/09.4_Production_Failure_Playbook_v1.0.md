## 一、故障分类与处置原则

| 故障类型 | 表现 | 处置原则 | 记录 |
|----------|------|----------|------|
| **Entry / Gate 失败** | Reject、Continue Chat、无 Execution | 不创建 Execution；用户侧提示；Trace Log 留痕 | gate_result、reason |
| **Context Load 失败** | Degraded Mode（空 Context） | 以空 Context 继续或中止；**仅只读/咨询型能力**，禁止 side_effect；审计记 context_load_failed；见 06.5 § 二 | Trace / Intent Log |
| **Intent 解析失败** | 非法 JSON、Intent 不在 Whitelist | task 不创建；n8n 错误处理；用户提示「未能理解意图」 | 06.5 |
| **Capability 超时** | Router 捕获 status=failed | audit 写入 timeout；不自动重试（除非 Policy 定义） | Exec Log、failure_mode |
| **Capability 缺 Memory** | failure_mode=missing_required_memory | Router 写入 audit；按 Policy 回退或提示 | Exec Log |
| **外部限流** | failure_mode=external_rate_limited | 记录 audit；可选重试或用户提示 | Exec Log |
| **pending_confirmation 超时** | status=expired | 不触发 Router；记录审计；不重试 | task_executions、02.4 |

---

## 二、处置流程（最小集）

1. **发现**：监控/告警或用户反馈；用 execution_id（或 correlation_id）查 Audit Trail（09.2）、回放（09.3）。
2. **定位**：按 failure_mode、gate_result、error 定位到 Entry / Router / Capability 层。
3. **处置**：按上表原则（不掩盖、不越权）；需修数据时走变更流程，不直接改生产审计表。
4. **闭环**：缺陷/故障记录与根因；与 Lily 协作补充用例或 Playbook 更新。

## 四、Lily 补充：QA 视角故障场景与复现步骤

以下场景基于 Entry Module 回归与 Gate 边界测试（EM-T4/T5/EM-T6、Lily-Alex 测试报告、Lily-Cat EM-T6 云端执行反馈），供故障注入与回归验证使用。复现方式：对 `POST /entry-adapter-webhook` 使用下表「复现输入」作为 Body（Content-Type: application/json），观察最终节点与输出是否与「预期表现」一致。

| 故障场景 | 复现输入要点 | 预期表现（最终节点 / 输出要点） | 可参考 |
|----------|--------------|--------------------------------|--------|
| **Structured Gate 失败：缺 intent** | entry_type=structured，payload={} 或 无 intent/intent_name | step1b_StructuredGateFail；_gate_pass: false，reason: missing_intent；**无** step1b_SAM_TaskCreate | EM-T4/T5 TC-S2、09.4 表 Entry/Gate 失败 |
| **Timeline Gate 失败：未到期** | entry_type=timeline，scheduled_at 为**未来**时间 | step1c_TimelineGateFail；_gate_pass: false，reason: not_expired；**无** step1c_SAM_TaskCreate | EM-T4/T5 TC-T2 |
| **非法 entry_type** | entry_type=unknown_type（或其它非 raw_query/structured/timeline） | step1d_UnknownEntryType；_unknown_entry_type: true，message: "Unsupported entry_type."；**不**调用任一 Sealer | EM-T4/T5 TC-U |
| **raw_query 空/缺字段 → Continue Chat** | raw_query="" 或 缺 raw_query 字段 | step11_ContinueChat；gate_result: continue_chat，reason（如 capability_not_found）；**无** Execution 写入 | EM-T6 GB-U3 |
| **幂等第二次返回** | 同一 idempotency_key 的 Body **连续调用两次** | 第一次：Sealer 返回 success + execution_id；第二次：同 execution_id 或 message 含 "Idempotent request: returned existing Execution." | EM-T6 GB-S3、GB-T3 |

### 4.1 复现输入示例（可直接复制）

**Structured Gate 失败（缺 intent）**：

```json
{"entry_type":"structured","entry_channel":"admin_os","requester_id":"user_test_002","payload":{},"meta":{}}
```

**Timeline Gate 失败（未到期）**：

```json
{"entry_type":"timeline","entry_channel":"system","requester_id":"system_cron","payload":{"intent":"publish_single","intent_name":"publish_single","timeline_id":"tl_test_002","scheduled_at":"2030-12-31T23:59:59.000Z","slots":{}},"meta":{"timeline_id":"tl_test_002"}}
```

**非法 entry_type**：

```json
{"entry_type":"unknown_type","entry_channel":"admin_os","requester_id":"user_test","payload":{}}
```

**raw_query 空字符串 → Continue Chat**：

```json
{"entry_type":"raw_query","entry_channel":"admin_os","requester_id":"user_test_u3","raw_query":"","meta":{"chat_id":"test_u3"}}
```

### 4.2 验证要点（QA 检查项）

- 上述故障场景下**无** Execution 写入（或仅幂等返回已有 Execution）；Trace/审计中可查到 gate_result、reason 或 _unknown_entry_type。
- 与 §一 故障分类、§二 处置流程一致；发现新故障表现时可反哺 Playbook 与 Cat 同步更新。
