## 一、n8n 只做编排

- **职责**：多端输入适配、从 DB 读取 Prompt、流程可视化编排、按 entry_type 分流、调用 SAM API（Context Load、Gate、Sealer）、调用 Router/Capability 实现（如 WF 绑定）。
- **不负责**：业务正确性判定、Task/Execution 持久化（由 SAM/DB）、Router 决策逻辑（由 Code Node 或确定性子流）、Capability 内部业务实现。

---

## 二、边界与约束

| 约束 | 说明 |
|------|------|
| **不新建 Gateway** | Router 即调度核心；n8n 中 Router 子流/节点只做「调用 Capability、写 audit」，不插入额外网关层 |
| **EntryRequest 归一化** | Entry Adapter（step0）输出符合 EntryRequest 的字段；entry_type、idempotency_key、requester_id 等由 n8n 或上游保证 |
| **Intent Analyzer 输出** | LLM 节点输出仅限 intent + slots；严禁输出 capability_chain；解析失败或非法 JSON 按 06.5 回退 |
| **Sealer 契约** | 调用 SAM Task Create 时传入 entry_type、requester_id、intent_name、payload、审计字段；不篡改契约字段 |
| **Capability 调用** | 通过 Capability Registry 与 implementation_binding 调用（如 WF15）；Envelope + Audit 必须满足 Integration Spec |

## 三、允许场景（n8n 实现视角）

> **来源**：Alex 基于 EM-T2/T4/T5 工作流梳理

| 场景 | n8n 实现 | 参考工作流/节点 |
|------|----------|-----------------|
| **Entry Adapter** | 多端输入（Webhook/Telegram/Manual）归一化为 EntryRequest；entry_type 分流 | step0_EntryAdapter；`EM-T2_Entry_Adapter_AllPaths_Workflow.json` |
| **简单控制流** | IF 分流（raw_query / structured / timeline / unknown）；Gate 通过/失败分支 | step1_IfRawQuery、step1b_IfStructured、step1c_IfTimeline、step10_IfGatePass |
| **Context Load 调用** | HTTP 调用 SAM `/api/intent/context/load`；传入 chat_id、user_id、raw_query | step3_SAM_ContextLoad |
| **Intent Analyzer** | LLM 节点输出 intent + slots；Code 解析为 Intent IR；**仅限 intent/slots，不输出 capability_chain** | step7_LLM_Intent、step8_ParseIntentIR |
| **Entry Gate 调用** | HTTP 调用 SAM `/api/entry/gate/check`；根据 gate_result 分流 | step9_SAM_GateCheck、step10_IfGatePass |
| **Sealer 调用** | HTTP 调用 SAM `/api/intent/task/create`；获得 execution_id 后触发 Router | step11_SAM_TaskCreate、step1b_SAM_TaskCreate、step1c_SAM_TaskCreate |
| **简化 Gate** | Code 节点实现基础合规、到期校验、防重锁（Spec 1.5）；不涉及 LLM 决策 | step1b_StructuredGate、step1c_TimelineGate |
| **非核心定时任务** | Cron 触发、按 schedule 扫描；payload 由上游提供，经 structured/timeline 路径 | 可挂接在 Entry Adapter 上游 |

## 四、禁止场景（n8n 实现视角）

> **依据**：FSD 05.2 Forbidden Intelligence、Integration Spec、08.2 Router 职责

| 禁止项 | 说明 | n8n 侧约束 |
|--------|------|------------|
| **LLM 直接调度 Router** | Router 必须 Code 实现；禁止 LLM 输出 capability_chain 或路由决策 | Intent Analyzer 输出仅限 intent + slots；step8_ParseIntentIR 严禁映射或输出 capability_chain |
| **直写核心表** | tasks、task_executions 由 SAM Sealer 写入；n8n 不直接 INSERT/UPDATE | 禁止 Supabase 节点写 tasks、task_executions；仅通过 HTTP 调 Sealer API |
| **绕过 Gate 调 Sealer** | Gate 不通过时不得创建 Execution | step10_IfGatePass、step1b_IfGatePass、step1c_IfGatePass 必须正确分流；False 分支不接 Sealer 节点 |
| **绕过 Sealer 调 Router** | Router 只接收 execution_id；无 execution_id 不得触发 Router | 必须先调 Sealer 获得 execution_id，再传递 execution_id 触发 Router |
| **模型自判风险等级** | 风险等级来自 Policy/Code，禁止 LLM 推断 | Intent Analyzer 不输出 risk_level、policy_decision 等；Gate 判定由 SAM 或 n8n Code 实现 |
| **在线黑箱学习** | Evolution 仅通过 Recommendation；禁止运行时改写 Router | n8n 中无 Evolution 直接改写逻辑；Recommendation 仅由 Router 接收与审计 |

## 五、n8n 与 Router 的边界

> **入口**：n8n 传递给 Router 的仅为 **execution_id**  
> **出口**：Router 返回结果/audit；n8n 不解析 Router 内部决策，仅传递 execution_id 与接收下游响应

| 边界点 | n8n 侧 | Router 侧 |
|--------|--------|-----------|
| **入口** | Sealer 返回 execution_id 后，n8n 将 execution_id 传递给 Router（如 Webhook 触发、Execute Workflow、HTTP 调用） | Router 只接收 execution_id 作为唯一入口；不接收 task_id、raw_query、intent 等 |
| **出口** | n8n 可接收 Router 返回的 CapabilityOutput、audit 摘要，用于返回用户或写入日志 | Router 返回 result、evidence、audit；n8n 不篡改 audit 内容 |
| **审计贯穿** | n8n 透传 entry_type、requester_id、intent_name 至 Sealer；不伪造审计字段 | audit 由 Router/Capability 写入；execution_id 贯穿全程 |

**实现参考**：`EM-T2_Entry_Adapter_AllPaths_Workflow.json` 中 step11_SAM_TaskCreate、step1b_SAM_TaskCreate、step1c_SAM_TaskCreate 输出 `execution_id`；下游 Router 子流/节点以 `execution_id` 为入参调用。

## 六、审计与可观测性要求

| 要求 | n8n 侧实现 |
|------|------------|
| **used_skills 声明** | 工作流 JSON `meta.used_skills` 声明使用的 Skill；与 Lucy 审计一致 |
| **节点可追溯** | 节点命名遵循 Step0_/StepN_ 规范；关键节点 notes 含资产来源 |
| **Gate 失败留痕** | step11_ContinueChat、step1b_StructuredGateFail、step1c_TimelineGateFail 输出 reason、message；不写入 Execution |
| **Sealer 契约不篡改** | 调用 Sealer 时传入的 entry_type、requester_id、intent_name、payload、metadata 与 Sam-Alex 契约一致 |
