## 一、加载来源与顺序

| 来源 | 路径 | 说明 |
|------|------|------|
| **项目级** | `.cursor/skills/<skill-name>/SKILL.md` | Cursor 启动时扫描；Agent 按对话上下文与 description/When to Use 判断是否采用 |
| **用户级** | `~/.cursor/skills/<skill-name>/SKILL.md` | 可选；与项目级合并，不覆盖同 ID |
| **规范源（Lucy 库）** | `Knowledgebase/Lucy_n8nWorkflowAuditor/2.Knowledge/Cursor_Agent_Skills/<skill-name>/SKILL.md` | 版本管理与审计用；**使用前须复制到 .cursor/skills/** 方能被 Cursor 加载 |

- **唯一准入**：所有进入 `.cursor/skills/` 的 Skill 由 Lucy 设计、编写、优化、部署或审核后纳入；Lucy 在每次产出或审计后更新 Skill_Registry。

---

## 二、门控规则（何时采用某 Skill）

| 规则 | 说明 |
|------|------|
| **上下文嗅探** | Agent 根据当前对话中的显式请求、关键词、路径/文件名与各 SKILL.md 的 **description**、**When to Use** 匹配；匹配则注入该 Skill 的 Instructions |
| **不触发** | 若 SKILL.md 中写明「不触发」条件（如「仅阅读文档、不产出评审结论时不触发」），则对应场景不采用 |
| **显式调用** | 用户使用 `/skill-name` 或明确说「用 xxx skill」时，优先采用该 Skill（若存在且合规） |
| **disable-model-invocation** | 若为 true，仅当显式调用时注入，不参与自动嗅探 |

---

## 三、合规门控（与 Registry 一致）

| 检查项 | 说明 |
|--------|------|
| **Registry 在册** | 在工作流/Agent/Prompt 中**正式引用**的 Skill 必须在 Skill_Registry 中有一行且 _compliance_status 为 compliant（或团队约定的等效状态） |
| **未注册不得引用** | 未在 Registry 登记的 Skill 不得作为正式契约写入 FSD/Integration Spec/工作流 JSON 的 used_skills |
| **抽检** | LEE (Memory & Compliance Manager) 可对 Registry 与引用处做非侵入式抽检；违规时 LEE 生成合规纠偏报告并通知 |
