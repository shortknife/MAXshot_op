# 07.5 捆绑式管理空间 v1.0

> **牵头**：Lucy (Skill Architect)  
> **依据**：Skill Registry、07.3 / 07.4  
> **FSD 位置**：`07_Skills_Compatibility/`  
> **日期**：2026-02

---

## 一、定义

**捆绑式管理空间**：与 MAXshot 仓库绑定的、由 Lucy 统一治理的 Skill 规范源与运行副本的约定关系。

---

## 二、空间划分

| 空间 | 路径 | 用途 | 维护 |
|------|------|------|------|
| **规范源（Lucy 库）** | `Knowledgebase/Lucy_n8nWorkflowAuditor/2.Knowledge/Cursor_Agent_Skills/<skill-name>/SKILL.md` | 版本管理、审计、留痕；唯一权威副本 | Lucy |
| **运行副本（Cursor 加载）** | `.cursor/skills/<skill-name>/SKILL.md` | Cursor Agent 实际扫描与加载 | 从规范源复制或软链；Lucy 产出/审计后同步 |
| **登记表** | `Knowledgebase/Lucy_n8nWorkflowAuditor/2.Knowledge/Skill_Registry.md` | 准入、合规状态、审计时间戳 | Lucy |

- **若适用**：当前 MAXshot 采用「规范源 + 运行副本」二分；未采用用户级全局 Skill 包或额外 extraDirs 时，以项目内 `.cursor/skills/` 为唯一运行空间。

---

## 三、同步约定

| 动作 | 约定 |
|------|------|
| **新增/变更 Skill** | Lucy 在规范源编写或修改 SKILL.md → 更新 Skill_Registry → 复制到 `.cursor/skills/<skill-name>/SKILL.md`（或执行 Skill_生成记录 中的部署命令） |
| **Reload** | 修改运行副本后，Cursor Reload Window 或重启以重新扫描 |
| **审计后** | Lucy 审计某 Skill 后更新 Registry 该行 _compliance_status、_audit_timestamp；若内容有变更须同步规范源与运行副本 |

---

## 四、与 OpenClaw 的差异（若集成）

- OpenClaw 侧可配置 `skills.load.extraDirs` 指向**只读**的 MAXshot 子路径（如 `.cursor/skills/` 或导出的 skill pack），实现「OpenClaw 加载 MAXshot 登记 Skill」；见 07.1。
- MAXshot 主仓库内仍以「规范源 + 运行副本」为捆绑式管理空间，OpenClaw 为消费方。
