## 一、失败场景与行为

| 场景 | 行为 | 回退/用户侧 |
|------|------|-------------|
| **LLM 输出非合法 JSON** | task 不创建 | n8n 抛出错误并记录日志，返回用户「未能理解意图」；可重试或 Continue Chat |
| **Intent 不在 Whitelist** | task 不创建 | 触发兜底意图或提示用户可用功能列表；不进入 Sealer |
| **Intent Analyzer 溢出 / LLM 调用失败** | 不 Create Execution | **回退路径**（按优先级）：(1) 明确失败提示 + 建议用户重试或简化表述；(2) 若已配置关键词匹配回退，则走关键词匹配产出 intent（仅限只读类）；(3) 否则 Continue Chat。审计记录 intent_analyzer_failed、原因（overflow / llm_error）。 |
| **Context Load 失败** | 进入 **Degraded Mode** | 以空 Context 继续流程，仅依赖当前 Query 进行分析；审计记录 context_load_failed；**能力上限**见下 § 二 |
| **Entry Gate 不通过** | 不 Create Execution | Continue Chat 或 Reject（依产品原则）；无 Task/Execution 写入，可审计 |

---

## 二、Degraded Mode 能力上限（宪法级）⭐

> **目的**：防止「既然还能继续，那能不能顺便执行一下？」——Degraded 时**能力必须受限**。

| 约束 | 必须写死 |
|------|----------|
| **禁止高风险 Capability** | Degraded Mode 下**禁止**触发 `risk_class=side_effect` 或需 Confirmation 的 Capability |
| **仅只读/咨询型** | Degraded Mode **只能**进入只读（如 data_fact_query、product_doc_qna）或咨询型能力 |
| **禁止写入** | 不创建/更新 Task、Execution、Memory；不执行发布、发送等 side_effect |

**落地**：Router 在检测到 `context_load_failed` 或等效标记时，Capability Chain 必须被**白名单过滤**，仅允许 Policy 定义的「Degraded 允许能力列表」。

---

## 三、与 06.1–06.4 的衔接

- **06.1**：Intent Analyzer 为 Orchestration 层角色；失败时不越权输出 capability_chain。
- **06.2–06.3**：EntryRequest / Session Context 缺失或非法时，按 Gate 与 Context Load 契约处理（Reject 或 Degraded）。
- **06.4**：Prompt 设计需考虑「未知意图」「解析失败」等输出形状，便于 n8n 分支处理。
- **§四 模型降级**：与 §一「LLM 调用失败」配合，先尝试 fallback model，再走回退路径。

---

## 四、模型降级路径（MVP 架构补全）⭐

- **目的**：Intent Analyzer 依赖的 LLM 服务不可用时，避免整条 Ops 主链路不可用。
- **策略**：配置 **fallback model**（如主模型 Claude 3.5 Sonnet → 备选 GPT-4o 或同 Provider 其他模型）；主模型调用失败（超时、4xx/5xx、quota）时自动切换。
- **触发条件**：LLM 节点返回不可用或重试达上限后触发 fallback；若 fallback 仍失败则按 §一「Intent Analyzer 溢出/LLM 调用失败」回退。
- **能力差异披露**：若发生 fallback，应在审计中记录 `model_used`、`fallback_occurred`（可选），便于 Admin OS 展示与排障；产品可约定「降级时能力差异」的披露策略（如仅内部分析用）。

**落地**：n8n Intent Analyzer 工作流中 LLM 节点配置 fallback model；Integration Spec / 09 可观测性可扩展 audit 字段。

---

## 五、回退策略原则

- **不掩盖**：失败必须留痕（Trace/Intent/Exec Log），不伪装成成功。
- **不越权**：Intent Analyzer 不替代 Router 做任务分解；解析失败即止，不写入 Task/Execution。
- **可审计**：entry_type、requester_id、失败原因、timestamp 可追溯；与 FSD 02.4、03.3 一致。
