# 02.1 层级总览

> **用途**：建立系统分层心智模型

---

## 一、分层结构

```
┌─────────────────────────────────────────────────────────┐
│  Soul Layer（横切，无 API）                               │
├─────────────────────────────────────────────────────────┤
│  Entry Layer（Gate、封印、Reject）                        │
├─────────────────────────────────────────────────────────┤
│  Router Layer（Task Decomposition、Working Mind、编排）   │
├─────────────────────────────────────────────────────────┤
│  Capability Layer（能力执行；定义与接入见 **02.5**）       │
├─────────────────────────────────────────────────────────┤
│  Memory Layer（Atomic：Foundation / Experience / Insight）│
├─────────────────────────────────────────────────────────┤
│  Evolution Engine（归因、反思、Insight 沉淀）             │
└─────────────────────────────────────────────────────────┘
```

---

## 二、数据域划分（2026-02-21 新增）

| 域 | 含义 | 示例 |
|----|------|------|
| **Business Data Plane** | 业务数据面 | vault、execution、market、allocation facts |
| **Governance Data Plane** | 治理控制面 | task_executions、audit controls、confirm/replay/retry/expire |

**原则**：业务查询必须优先查 Business Data Plane；不得静默降级为仅查 Governance Data Plane。

---

## 三、数据流

- **Input** → Entry → (Reject / 封印) → Task + Execution
- **Router** ← execution_id → Task Decomposition → memory_refs + capability_chain
- **Capability** ← CapabilityInput（含 context.memory_refs）→ CapabilityOutput
- **Evolution** ← Execution 结果 → Attribution → Insight 写入 agent_memories

---

## 四、关键约束

- Entry 不判断「对不对」，只判断「能不能负责执行」
- Router 只认 execution_id，不管理 Task
- Capability 不直接访问 DB（通过 Router/context 传递）
- Evolution 对 Router 只能通过 Recommendation 影响

---

## 五、产品决议（本版唯一执行）

**决议 2（Gate 双重角色）**：Entry Gate（放行/拒绝进系统、价值信息判断）与 Capability Gate（G1–G6、contract_passed=false 不返回 success）须在文档与实现中显式区分，避免混用「Gate」。  

**决议 6（业务数据优先）**：§二 数据域划分为准；主产品查询答案仅来自 Business Data Plane；验收含数据源白名单与负面用例（答出 task_executions/audit 则失败）。  

**决议 7（Capability 不直连 DB）**：Capability 数据仅通过 Router/context（含 memory_refs）获取，不直连 DB；§四 关键约束已体现。  

**开发/验收要点**：开票「业务查询」「Data Query」时须对齐 Business 表清单；Capability 开发标准写明不直连 DB。**Capability 定义、列表与接入**见 **02.5**（`02.5_Capability_Definition_And_Registry.md`）。
