# 00.4 如何阅读本文档集

> **用途**：新同学阅读路径与文档关系说明

---

## 一、阅读顺序（建议）

1. **00 Read First**（本目录）— 必读，建立共识
2. **01 User Journey** — 理解拒绝路径与安全设计
3. **02 Layer Model** — 理解谁决策、谁审计、记在哪
4. **03 Execution Model** — 理解 Task/Execution/Snapshot
5. **04 Working Mind** — 理解心智合成与 Evolution
6. **05 Intelligence Boundary** — 理解 Allowed/Forbidden/Grey
7. **06–10** — 按需深入（Intent Analyzer、Skills、Integration、Observability、Appendix）

---

## 二、与产品架构的关系

本 FSD 文档集是 **v5.1 产品架构** 的执行级规格（继承 v5.0 分层与宪法，v5.1 做务实收敛）。

- **当前执行版（权威）**  
  → `FSD/MAXshot_产品架构设计_v5.1_Small_Team_Growth_OS.md`

- **愿景参考（冻结）**  
  → `FSD/MAXshot_产品架构设计_v5.0_Autonomous_Soul.md`

- **本 FSD**  
  → 回答「怎么拒绝、怎么审计、怎么回放、怎么不被自己坑」

## 三、文档状态

- **v5.0 核心原则**：冻结，不推翻（Router 确定性、Execution 不可变等）
- **v5.1 执行版**：本 FSD 以 v5.1 为执行权威，分层继承 v5.0，Evolution/Soul/Memory 按 v5.1 收敛描述
- **本版唯一执行**：**仅执行现有 8 条产品决议**；产品设计优劣全部放入下一版本，本版不反复改需求。8 条决议的可执行细节已融入 01–09 各 Module，每 Module 内设「产品决议（本版唯一执行）」小节，供开发与验收核对。
