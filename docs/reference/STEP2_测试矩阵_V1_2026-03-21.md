# Step 2 测试矩阵 V1

- Status: Frozen

## 目的

本测试矩阵只验证 `Step 2 - Conversation Context Manager`。  
不验证 Step 3 capability match，不验证最终业务结果。

测试对象是：
1. `session_resolution`
2. `turn_relation`
3. `conversation_context`
4. `registry_context`
5. `memory_runtime`
6. `context_policy`

---

## 测试字段模板

每条测试都记录：
- `Test ID`
- `原始输入`
- `前置 session 状态`
- `期望 turn_relation`
- `期望 session_action`
- `期望 context policy`
- `是否应触发 recall`
- `失败归因`

---

## A. Session 基础类

### T2-A1 无 session 的独立输入
- 输入：`当前 vault APY 怎么样？`
- 前置状态：无
- 期望：
  - `turn_relation = new_session`
  - `thread_action = continue` 或 `reset`（按最终 contract 固定）
  - 不触发 recall

### T2-A2 有 session 的普通续问
- 输入：`那 arbitrum 呢？`
- 前置状态：
  - 上轮为 `yield`
  - 最近 7 天
- 期望：
  - `turn_relation = continuation`
  - 继承时间范围

### T2-A3 明确 reset
- 输入：`/new 当前 vault APY 怎么样？`
- 前置状态：存在旧 session
- 期望：
  - `turn_relation = new_session`
  - `thread_action = reset`

---

## B. Clarification 类

### T2-B1 标准澄清回复
- 输入：`最近7天`
- 前置状态：
  - pending clarification 缺 `time_window`
- 期望：
  - `turn_relation = clarification_reply`
  - 只补缺槽

### T2-B2 澄清期间切新话题
- 输入：`你能描述什么是MAXshot么？`
- 前置状态：
  - pending clarification 仍存在
- 期望：
  - `turn_relation = new_topic_same_window`
  - 清空旧 clarification

---

## C. Correction 类

### T2-C1 更正 scope
- 输入：`不对，我问的是 vault 不是 market`
- 前置状态：
  - 上轮 scope = market
- 期望：
  - `turn_relation = correction`
  - 旧 scope 被覆盖

### T2-C2 更正时间
- 输入：`不对，我要看最近30天`
- 前置状态：
  - 上轮 time_window = 7d
- 期望：
  - `turn_relation = correction`
  - 继承其他上下文，覆盖 time_window

---

## D. New Topic 类

### T2-D1 同会话新话题
- 输入：`写一条关于新品发布的帖子`
- 前置状态：
  - 上轮为业务 APY 查询
- 期望：
  - `turn_relation = new_topic_same_window`
  - 业务上下文清空

### T2-D2 产品定义问题切入
- 输入：`MAXshot 品牌故事是什么？`
- 前置状态：
  - 上轮为 vault 查询
- 期望：
  - `turn_relation = new_topic_same_window`
  - 不继承 APY/vault filters

---

## E. History Callback 类

### T2-E1 显式回溯历史对象
- 输入：`我之前问的 dForce 那个再看一下`
- 前置状态：
  - session 内存在 dForce 查询历史
- 期望：
  - `turn_relation = history_callback`
  - 触发 recall

### T2-E2 回到刚才那个 vault
- 输入：`回到刚才那个 vault`
- 前置状态：
  - 最近上下文存在明确 vault
- 期望：
  - `turn_relation = history_callback`
  - 触发 recall

---

## F. Registry / Memory Runtime 类

### T2-F1 active capability registry 装载
- 输入：任意业务问题
- 前置状态：正常运行时
- 期望：
  - `registry_context.active_capability_ids` 存在

### T2-F2 memory runtime 装载
- 输入：任意业务问题
- 前置状态：正常运行时
- 期望：
  - `memory_runtime.ref_ids` 存在
  - `memory_runtime.source_policy` 正确

---

## G. 不应发生的错误

以下情况一旦出现，Step 2 直接判不通过：

1. 新问题被识别成旧澄清补充
2. correction 被识别成 continuation
3. history callback 被当成普通 follow-up
4. `turn_relation` 缺失
5. `Context Envelope` 字段散落、不成结构
6. active capability registry 没被装载
7. memory runtime 缺失或无来源策略

## 通过标准

Step 2 通过的最低要求：

1. A/B/C/D/E/F 各组测试都有明确输出
2. relation taxonomy 不缺项
3. 每种 relation 至少 1 正例通过

5. 旧上下文污染类错误被消除
