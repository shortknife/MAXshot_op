## 第一层：业务规格

### 1.1 产品概述

#### 产品定位
Nexa是一个**B2B SaaS智能FAQ Bot平台**，基于X402支付协议和RAG技术，为项目方提供智能FAQ服务基础设施，让项目方能够为终端用户提供高质量的FAQ问答服务。

**产品Slogan**：Connect Knowledge, Instant Answers（连接知识，即时回答）  
**品牌调性**：现代、智能、连接、未来

#### 目标用户

**主要用户（B端客户）**：
- **项目方（企业用户）**：需要为终端用户提供FAQ服务的项目方
- **使用场景**：产品咨询、技术支持、售后支持
- **付费方式**：按问答次数付费（X402协议）

**终端用户（C端用户）**：
- **项目方的用户**：使用项目方提供的FAQ服务
- **使用方式**：免费使用FAQ服务
- **接入方式**：网页端、Telegram Bot、Discord Bot、X（Twitter）被动回复

#### 核心价值

**对项目方**：
1. **降低FAQ服务成本**：无需自建FAQ系统，按需付费
2. **提升服务质量**：基于RAG技术的智能问答，回答更准确、自然
3. **多端统一服务**：一次配置，多端部署（网页/TG/DC/X）
4. **区块链原生支付**：X402协议，微支付，降低使用门槛

**对终端用户**：
1. **免费使用**：无需付费即可获得FAQ服务
2. **多端接入**：通过熟悉的平台（网页/TG/DC/X）使用服务
3. **即时响应**：智能问答，快速获得答案

#### 产品差异化

| 维度 | 传统FAQ Bot | 现代LLM/RAG FAQ Bot | **我们的产品** |
|------|------------|---------------------|--------------|
| **商业模式** | 订阅制（按月/年） | 订阅制（按月/年） | ✅ **X402按次付费** |
| **用户角色** | 单一角色（企业自用） | 单一角色（企业自用） | ✅ **双角色（B2B SaaS）** |
| **内容范围** | 可回答范围较宽泛 | 可回答范围较宽泛 | ✅ **严格限制在知识库范围内** |
| **边界控制** | 转人工或模糊回答 | 可能有幻觉问题 | ✅ **双重检测+明确拒绝** |
| **支付方式** | 传统支付（信用卡等） | 传统支付（信用卡等） | ✅ **区块链原生支付（X402）** |

**核心创新点**：
1. **严格的内容边界控制**：仅回答知识库内容，避免幻觉
2. **智能超出范围检测**：双重判断（向量+LLM），避免无效收费
3. **区块链原生支付**：X402协议，微支付，降低门槛
4. **B2B SaaS模式**：项目方付费，终端用户免费

#### 2.2.3 平台核心功能

##### F6. FAQ问答引擎（RAG）
**功能描述**：基于RAG技术的智能问答引擎

**功能详情**：
- **F6.1 向量检索**
  - 检索方式：基于Embedding的语义相似度搜索
  - 检索数量：Top 3相关文档片段
  - Embedding模型：Gemini Embeddings

- **F6.2 答案生成**
  - 生成方式：RAG（Retrieval-Augmented Generation）
  - LLM模型：DeepSeek
  - 生成要求：仅基于知识库内容生成答案，不自由发挥

- **F6.3 答案质量控制**
  - 置信度判断：基于向量相似度和LLM判断
  - 置信度阈值：0.85（与Telegram Chatbot一致）
  - 质量控制：确保答案不超出知识库范围

**验收标准**：
- ✅ 向量检索能准确找到相关文档片段
- ✅ RAG生成的答案准确、自然
- ✅ 答案不超出知识库范围
- ✅ 置信度判断准确

##### F7. 超出范围检测机制
**功能描述**：双重检测机制，确保准确识别超出范围的问题

**功能详情**：
- **F7.1 第一层检测：向量检索**
  - 检测方式：快速向量检索
  - 检测标准：向量相似度 < 0.7 → 直接拒绝
  - 检测成本：免费（仅向量检索）

- **F7.2 第二层检测：免费LLM判断**
  - 检测方式：免费/低成本LLM判断（如DeepSeek）
  - 检测标准：LLM判断"不在范围内" → 拒绝
  - 检测成本：免费/低成本
  - 检测时机：向量相似度 ≥ 0.7时进行

- **F7.3 检测结果处理**
  - 在范围内：进入支付流程 → RAG回答
  - 超出范围：拒绝回答，不收费

**验收标准**：
- ✅ 双重检测机制能准确识别超出范围的问题
- ✅ 超出范围的问题不收费
- ✅ 检测响应时间 < 2秒

  - 处理方式：不提供答案，提示项目方检查余额

**验收标准**：
- ✅ 支付流程正确执行

## 第三层：交互规格

### 3.1 核心用户流程

#### 3.1.1 项目方注册和知识库上传流程

```
1. 项目方访问平台
2. 点击"连接钱包" → WalletConnect连接
3. 绑定钱包地址 → 创建账户
4. 进入Dashboard
5. 点击"上传知识库"
6. 选择Markdown文档 → 上传
7. 系统自动处理（切片、向量化、存储）
8. 上传成功 → 显示知识库预览
```

**边界情况**：
- 钱包连接失败 → 提示重试
- 文档格式错误 → 提示格式要求
- 上传失败 → 提示错误原因

---

#### 3.1.2 终端用户FAQ问答流程（网页端）

```
1. 终端用户通过API调用FAQ服务
2. 系统接收问题
3. 第一层检测：向量检索
   - 相似度 < 0.7 → 拒绝，不收费
   - 相似度 ≥ 0.7 → 进入第二层检测
4. 第二层检测：免费LLM判断
   - 判断"不在范围内" → 拒绝，不收费
   - 判断"在范围内" → 进入支付流程
5. 支付流程：
   - 检查项目方余额
   - 余额不足 → 提示支付失败
   - 余额充足 → 执行支付
6. RAG生成答案
7. 返回答案给终端用户
8. 记录问答历史
```

**边界情况**：
- 问题超出范围 → 明确拒绝，不收费
- 余额不足 → 提示支付失败
- 支付失败 → 提示检查余额
- 答案生成失败 → 提示错误

---

#### 3.1.3 终端用户FAQ问答流程（X端）

```
1. 系统监控X端Mentions和回复
2. 检测到新问题
3. 双重检测（同网页端流程）
4. 支付流程（同网页端流程）
5. RAG生成答案
6. 检查答案长度：
   - ≤ 280字符 → 直接回复
   - > 280字符 → 截断核心内容 + 引导链接
7. 发布回复到X
8. 记录问答历史
```

**边界情况**：
- 答案超出280字符 → 截断 + 引导链接
- 回复失败 → 记录错误日志

---

### 3.2 超出范围处理流程（详细设计）

#### 3.2.1 检测流程

```
用户提问
    ↓
第一层检测：向量检索（免费）
    ↓
相似度 < 0.7？
    ├─ 是 → 拒绝，不收费，提示："抱歉，这个问题不在知识库范围内。请尝试其他问题。"
    └─ 否 → 进入第二层检测
         ↓
第二层检测：免费LLM判断（免费）
    ↓
LLM判断"不在范围内"？
    ├─ 是 → 拒绝，不收费，提示："抱歉，这个问题不在知识库范围内。请尝试其他问题。"
    └─ 否 → 进入支付流程
         ↓
支付流程 → RAG回答
```

#### 3.2.2 提示文案设计

**超出范围提示文案**：
- 中文："抱歉，这个问题不在知识库范围内。请尝试其他问题。"
- 英文："Sorry, this question is outside the knowledge base. Please try another question."

**设计原则**：
- 明确告知超出范围
- 不提供模糊选项
- 引导用户尝试其他问题
- 保持友好、专业的语气

---

### 3.3 支付流程（详细设计）

#### 3.3.1 支付时机和流程

```
用户提问
    ↓
双重检测通过（在范围内）
    ↓
检查项目方余额
    ├─ 余额不足 → 提示："余额不足，请充值后重试"
    └─ 余额充足 → 继续
         ↓
生成答案（RAG）
    ↓
返回答案给用户
    ↓
执行支付（X402协议）
    ├─ 支付成功 → 记录交易，更新余额
    └─ 支付失败 → 提示："支付失败，请检查余额"
```

**关键设计点**：
- **支付时机**：问答后支付（用户已获得答案）
- **支付失败处理**：提示用户，但不撤回答案（用户体验优先）
- **余额检查**：支付前检查，避免无效支付
