## 一、概述

本文档提供了 Intent 相关的 Skill 示例，展示如何在实际工作流中使用 Prompt 设计 Skill 来构建 Intent Analyzer。

### 1.1 Skill 使用场景

| Skill 名称 | 触发条件 | 使用场景 | Gateway |
|-----------|---------|---------|---------|
| `prompt/four-layer-prompt-architecture` | Prompt 设计、优化 | Intent Analyzer Prompt 设计 | `four_layer_prompt_executor` |

### 1.2 Intent Analyzer 工作流中的 Skill 调用链

```
用户查询 → Intent Analyzer (n8n 工作流)
  → Code 节点: 从 Supabase 读取 Prompt
    → Gateway: four_layer_prompt_executor (可选，用于 Prompt 设计验证)
  → Claude/OpenAI 节点: 执行 Prompt
  → 输出: 结构化 Intent JSON
```

## 三、n8n 工作流中的 Skill 使用模式

### 3.1 Code 节点：从 Supabase 读取 Prompt

```javascript
// Code 节点：从 Supabase 读取 Intent Analyzer Prompt
const { data: prompt, error } = await supabase
    .from('prompt_library')
    .select('*')
    .eq('slug', 'intent_analyzer_v2')
    .single();

if (error) {
    throw new Error(`Failed to load prompt: ${error.message}`);
}

return {
    json: {
        system_prompt: prompt.system_prompt,
        user_prompt_template: prompt.user_prompt_template,
        model_config: prompt.model_config
    }
};
```

### 3.2 Code 节点：变量注入和模板化

```javascript
// Code 节点：注入变量到 Prompt
const { system_prompt, user_prompt_template, model_config } = $input.item.json;

// 注入 Memory Layer Context
const memoryLayerContext = $input.item.json.memory_layer_context || '';
const systemPrompt = system_prompt.replace('{memory_layer_context}', memoryLayerContext);

// 注入 User Prompt 变量
const userPrompt = JSON.stringify({
    user_query: $input.item.json.clean_query,
    session_context: $input.item.json.session_context || {},
    vault_list: $input.item.json.vault_list || []
});

return {
    json: {
        system_prompt: systemPrompt,
        user_prompt: userPrompt,
        model_config: model_config
    }
};
```

## 四、Skill 执行流程（Cursor 中使用）

### 4.1 Skill Binding 声明

```markdown
## Skill Binding Declaration

- **Selected Skill**: `prompt/four-layer-prompt-architecture` (v1.0)
- **Binding Reason**: 用户请求设计 Intent Analyzer Prompt，需要使用四层架构方法
- **Binding Scope**: 整个 Intent Analyzer Prompt 设计流程
- **Execution Contract**: 已读取并理解，将严格遵守
```

### 4.2 执行步骤

**Step 1: 理解需求**
- 明确 Intent Analyzer 的使用场景：将自然语言查询转换为结构化 Intent JSON
- 确定输入输出要求：
  - 输入：用户查询、会话上下文、Memory Layer 上下文、金库列表
  - 输出：结构化 Intent JSON（intent、extracted_slots、context_inherited）

**Step 2: 设计 Persona Core 层**
- 定义角色：MAXshot Intent Analyzer
- 定义使命：解码自然语言为结构化执行参数
- 定义约束：不输出 capability_chain，不建议工具或工作流

**Step 3: 设计 Style & Voice 层**
- 输出风格：分析性、直接
- 格式要求：单行有效 JSON 对象
- 语言：英文键名，保留原始语义值

**Step 4: 设计 Behavior Rules 层**
- 意图分类：ops_query、marketing_gen、documentation、mixed、unknown
- 槽位归一化：vault_name、metric、time_range、is_fuzzy
- 上下文继承：使用 session_context 处理椭圆查询

**Step 5: 设计 Dynamic Context 层**
- 动态变量：memory_layer_context（合法 intent 描述 + 产品共识摘要）
- 上下文要求：会话状态、金库列表

**Step 6: 组合四层 Prompt**
- 按照标准格式组合四层 Prompt
- 注入动态变量，生成完整 Prompt

**Step 7: 验证和优化**
- 验证 token 范围：总计 850 tokens（合理）
- 验证变量映射：clean_query、session_context、memory_layer_context、vault_list
- 验证 Prompt 可执行性：符合四层架构要求

### 9.1 Skill 使用留痕


### 9.2 Prompt 设计合规性

所有 Prompt 设计文档必须通过以下验证：

- ✅ 四层架构完整性（四层是否齐全）
- ✅ Token 范围合理性（各层 token 分布是否合理）
- ✅ 变量映射正确性（变量名、格式、验证规则）
- ✅ Prompt 可执行性（是否能正确引导 LLM 完成任务）
