# 01.1 用户路径：Happy Path

> **用途**：描述标准成功路径，作为 Rejection Path 的对照

---

## 一、raw_query 路径

```
用户输入 (TG Bot / Admin OS Chat)
  → Entry Adapter 接收
  → Context Load（session_context snapshot）
  → Intent Analyzer（Soul 滤镜通过）
  → Sealer 写入 tasks + task_executions
  → (若 side_effect) pending_confirmation → 用户确认
  → Router 接收 execution_id
  → Task Decomposition + Working Mind Synthesis
  → Capability Chain 执行
  → 结果返回 + Audit 写入
```

---

## 二、structured 路径

```
Admin OS / Cron / Notion / System 输入
  → Entry Adapter 接收
  → Sealer 直接写入（跳过 Intent Analyzer）
  → Router 接收 execution_id
  → 执行
  → Audit 写入
```

---

## 三、timeline 路径

```
publishing_timeline 到期扫描
  → 防重锁（scheduled → publishing）
  → 到期校验（scheduled_at <= now()）
  → 封印 / 分发执行
  → Audit 写入
```

---

## 四、成功终态

- **Task**：active（scheduled）或 completed（ad_hoc）
- **Execution**：completed
- **Audit**：capability_chain、memory_refs、每步 status 已写入
