# 03.5 Task Families and No-Silent-Downgrade Addendum (2026-02-21)

> 类型：Addendum（补丁文件）  
> 作用范围：补充 `03.1`、`03.2`、`03.4`，不覆盖原文

## 1. Task Family 补充

- `business_query`：面向业务事实问答。
- `governance_action`：确认、运行、重放、重试、过期、写入门控。
- `content_operation`：内容生成、改写、反馈闭环。

## 2. 生命周期要求

- `business_query` 在 Outcome 中必须包含来源证据（RPC/模板/表）。
- 若来源缺失，Execution 视为失败或拒绝，不可标记成功。

## 3. 回放语义

- replay/retry 的输出应保持“治理动作结果”与“业务查询结果”分离展示。
- 回放成功不等于业务答案正确；业务答案正确性由 evidence 完整性判定。
