# 03.4 回放与审计流

> **用途**：说明如何基于 Execution 进行回放与审计

---

## 一、回放依据

- **execution_id** 是回放的主键
- 通过 `task_executions` + `audit_log` 可重建执行路径

---

## 二、审计流

```
Entry 决策 → Trace Log
  → Task/Execution 创建
  → Router 决策 → Intent Log
  → Capability 执行 → Exec Log（每步 audit）
  → 终态写入 task_executions.status
```

---

## 三、回放要求

- 所有 Log 必须包含 execution_id（或可关联到 execution_id）
- 支持按 execution_id 查询完整链路
