## 一、产品理由

- **单调度核心**：系统已有 **Router** 作为确定性调度中心（接收 execution_id、Task Decomposition、Capability 编排、审计写入）。再建独立 Gateway 层会重复职责、增加一致性与审计复杂度。
- **Entry 非 Gateway**：Entry 层负责「能否负责执行」（Gate 判定、封印、Reject、Continue Chat），不负责「往哪路由、调谁」；路由与编排权归属 Router。
- **Capability 直连 Router**：运行时只通过 Router 调用 Capability，不经过额外网关；Router 即唯一调度入口，便于审计与策略收敛。

---

## 二、与 FSD / 技术框架一致

- **FSD 02.2**：Router 负责接收 execution_id、Task Decomposition、Memory 选择、Capability 编排、审计写入；不负责 Task 生命周期。
- **Integration Spec**：Router 只接收 execution_id；运行时只调用 capability，绝不直连 skill / n8n workflow。
- **结论**：不新建 Gateway，**Router 即调度核心**；Entry 与 Router 边界清晰，技术架构全局判断以此为准。
