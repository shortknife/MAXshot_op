## 一、raw_query 路径

```
Entry (TG Bot / Admin OS Chat)
  → EntryRequest(entry_type=raw_query)
  → Entry Adapter → Context Load (SAM) → Intent Analyzer (LLM + Parse) → Entry Gate (SAM)
  → Gate 通过 → Sealer (SAM Task Create) → task_id + execution_id
  → Router(execution_id) → Task Decomposition → CapabilityInput → Capability (e.g. WF15)
  → CapabilityOutput → audit 写入 → Execution 状态更新 → 返回用户
```

- **Gate 不通过**：不调用 Sealer，Continue Chat 或 Reject；无 Execution。
- **Context Load 失败**：Degraded Mode（空 Context），见 06.5。

---

## 二、structured 路径

```
Entry (Admin OS / Cron / Notion / Webhook)
  → EntryRequest(entry_type=structured)
  → Entry Adapter → 简化 Gate (n8n/SAM)
  → Gate 通过 → Sealer (SAM Task Create) → task_id + execution_id
  → Router(execution_id) → … → Capability → audit → 返回/分发
```

- **不经过 Intent Analyzer**；payload 由上游提供 intent_name、slots 等。

---

## 三、timeline 路径

```
Entry (publishing_timeline 到期等)
  → EntryRequest(entry_type=timeline)
  → Entry Adapter → 到期校验 + 防重锁 → 简化 Gate
  → Gate 通过 → Sealer → execution_id
  → Router(execution_id) 或业务域分发 → …
```

- **防重锁**：同一 timeline 项不并发重复执行；到期校验 scheduled_at <= now()。

---

## 四、贯穿约束

- **Task/Execution 分离**：Router 只认 execution_id；Task 生命周期在 Entry/Orchestration。
- **审计贯穿**：entry_type、requester_id、intent_name、execution_id、capability_id、status、failure_mode 可追溯；与 03.3 不可变快照、09 Audit 一致。
- **Grey Area**：凡 Grey 一律 Require Confirmation（pending_confirmation），不交给 LLM 自判；见 FSD 05.3、05.4、Integration Spec 1.3.4。
