# 00.6 v5.1 Clarified Product Scope Addendum (2026-02-21)

> 类型：Addendum（补丁文件）  
> 作用范围：补充 `00.1`、`00.2`，不覆盖原文

## 1. 产品主次重新定义（执行口径）

1. 主产品（Primary）
- 面向客户/运营：自然语言查询 MAXshot 业务事实。
- 核心输出：基于业务数据的可追溯答案（含来源证据）。

2. 次产品（Secondary）
- 面向增长团队：营销内容生成、改写、反馈回流。

3. 底座（Control Plane）
- Admin 能力（confirm/run/replay/retry/expire/audit）是治理控制面，不是客户价值主入口。

## 2. 强约束

- 业务问题优先走业务数据面，不得以控制面日志替代业务答案。
- 无法获取业务数据时，必须显式拒绝，不得“看起来回答了”。
- 所有客户可见结论都要可追溯到数据来源。

## 3. 非目标补充

- 非目标：把系统执行日志当作业务洞察输出给客户。
- 非目标：仅围绕内部执行队列优化而忽视业务查询准确率。
