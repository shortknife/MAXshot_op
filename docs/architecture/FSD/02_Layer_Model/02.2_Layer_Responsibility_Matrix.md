# 02.2 层级职责矩阵

> **用途**：明确每层负责什么、不负责什么

---

## 一、职责矩阵

| 层级 | 负责 | 不负责 |
|------|------|--------|
| **Entry** | Gate 判定、封印、Reject、Continue Chat、创建 Task/Execution | 业务正确性、Intent 深度理解 |
| **Router** | 接收 execution_id、Task Decomposition、Memory 选择、Capability 编排、审计写入 | Task 生命周期、业务逻辑判断、错误修复 |
| **Capability** | 按契约执行、返回 result/evidence/audit | 路由决策、Memory 内容生成 |
| **Evolution** | 归因分析、策略闭环（v5.1：标签+归因+复盘；P3 后引入 Insight）、Recommendation 产出 | 直接改写 Router、在线学习 |

---

## 二、Soul（横切）

- **负责**：作为 Filter 介入 Entry/Intent/Content 的选择
- **不负责**：调度、执行、存储
