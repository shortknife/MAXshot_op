## 📌 备忘录：输入形态与 Intent Analyzer 调用边界（必读）

- **raw_query 输入**（Telegram / Chat UI 文本）→ **必走** Intent Analyzer
- **structured 输入**（定时任务、Notion 同步、后台按钮）→ **跳过** Intent Analyzer，直接封印
- **timeline 输入**（publishing_timeline 到期扫描）→ **跳过** Intent Analyzer，直接封印 + 防重锁

**Intent Analyzer 的被调用依据不是语义，而是通道/消息类型（raw_query vs structured）。**

**session_context ≠ memory_refs**：Intent Analyzer 的 session_context 用于指代消解，来自 tasks/对话历史；**不跨 turn 隐式演化**，在 `/context/load` 时生成 snapshot、turn 内复用；Router 的 memory_refs 由 Router 在任务分解时生成，用于 Capability 执行。

## 1. 架构定位 (Architectural Positioning)

**Intent Analyzer** 是系统 **Orchestration Layer (编排层)** 的核心逻辑角色（Logical Role）。

| 维度 | 说明 |
| :--- | :--- |
| **层级定位** | **Orchestration Layer** (高于 Execution Layer) |
| **上游入口** | Telegram Trigger / Webhook / UI Input |
| **下游交付** | `task_executions` 表 (写入 `payload` 字段) |
| **核心职责** | 将自然语言转化为结构化参数，并处理对话上下文（Session） |
| **宪法红线** | **严禁输出 `capability_chain`**。任务分解权归属下游的 Router。 |

---

## 2. 核心逻辑流程 (Core Workflow)

### 2.1 信号接收与清洗 (Input & Cleaning)
- 接收原始 Text 消息。
- 调用 `step2_CleanText` (资产 07) 进行脱敏与符号清洗。

### 2.2 上下文加载 (Context Loading)
- **Session 检查**：维持对话连续性。
- **意图继承 (Intent Persistence)**：处理指代消解（如"那 USDT 呢？"）。

### 2.3 意图分析与槽位提取 (NLU & Slot Extraction)
- **实现方式**：Phase 1 关键词匹配；Phase 2 升级为独立 LLM 节点。
- **LLM 边界 (Crucial Boundary)**：
    - **LLM 只做语义解析**：即将自然语言 → 结构化参数 (`intent` + `extracted_slots`)。
    - **LLM 绝不参与任务分解**：`capability_chain` 的决策权 100% 归属于 Router。
    - **输出契约冻结**：输出必须且仅包含 `intent` 和 `extracted_slots`。

### 2.4 数据交付 (Data Handover)
- 将识别出的 `intent` 和 `extracted_slots` 封装进 `payload` 字段。
- **Router 主权**：Router 接收 `execution_id` 后，基于 `payload` 内容自主进行任务分解。

---

## 3. 架构守卫 (Architecture Guardrails)

遵循 v4.1 宪法 2.3.1.1 节：
- **Router = 确定性调度器**：它是系统的治理中心。
- **LLM = 不可信建议源**：它仅作为意图提取工具，其输出必须经过 Router 的确定性逻辑过滤。

## 5. 验收标准 (Acceptance Criteria)

1. **确定性注入**：`task_executions` 表中的 `payload` 必须包含清晰的 `intent` 和 `extracted_slots`。
2. **无越权行为**：`payload` 中严禁出现 `capability_chain` 或任何指示 Router 如何工作的指令。
3. **上下文有效性**：当用户发送"USDC 呢？"时，`extracted_slots` 能够自动补全前序任务中的 `metric`。
