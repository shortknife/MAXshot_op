## 一、Log 类型

| 类型 | 用途 | 记录位置 | 必含字段 |
|------|------|----------|----------|
| **Trace Log** | Entry 决策、Gate 判定、Reject/Continue Chat | Entry 层（n8n/SAM） | execution_id（若已创建）、entry_type、requester_id、gate_result、timestamp |
| **Intent Log** | Router 任务分解、Intent 解析结果 | Router 层 | execution_id、intent_name、capability_chain、memory_refs_ref、timestamp |
| **Exec Log** | Capability 执行、每步 audit | Capability/Router | execution_id、capability_id、capability_version、status、failure_mode、elapsed_ms、timestamp |
| **Safety Log** | Grey Area、pending_confirmation、Reject 原因 | Entry/Orchestration | execution_id、reason、policy_ref、timestamp |

---

## 二、Schema 约束（最小集）

- **贯穿键**：所有 Log 必须可关联到 **execution_id**（或 entry 阶段为 correlation_id/request_id，创建 Execution 后补 execution_id）。
- **Exec Log**：与 Integration Spec `CapabilityOutput.audit` 一致；含 `execution_id`、`capability_id`、`capability_version`、`status`、`failure_mode`、`invocation_source`。
- **Trace Log**：Entry 层 Gate 不通过时可能尚无 execution_id；用 correlation_id 或 request_id 关联，Sealer 创建后写回 execution_id。
- **时间戳**：ISO8601；所有 Log 含 timestamp。

---

## 三、与 03.4、02.3 一致

- **03.4**：execution_id 是回放主键；task_executions + audit_log 可重建执行路径。
- **02.3**：谁决策、谁审计、记在哪；Entry 决策记 Trace，Router 记 Intent，Capability 记 Exec，Safety 事件记 Safety Log。
- **技术实现**：详见 Integration Spec §2.3 CapabilityOutput.audit、§8 宪法级条款落地点；A2 Schema Registry 列核心 Schema。
