## 一、回放依据

- **主键**：**execution_id**；通过 execution_id 查询可重建单次执行的完整链路。
- **数据源**：task_executions（Task/Execution 元数据、状态）、audit_logs（或等价 Exec Log / Trace Log），按 execution_id 关联。
- **不可变**：回放时读取的是 Execution 启动时的 Snapshot 及后续追加的审计记录，不 join 当前最新表（见 03.3）。

---

## 二、回放与调试流程

1. **按 execution_id 查询**：从 task_executions 获取 task_id、entry_type、requester_id、status、created_at；从 audit_logs 获取该 execution_id 下所有 audit 记录（按时间序）。
2. **重建链路**：Entry 决策（Trace）→ Sealer 创建 Execution → Router 任务分解（Intent）→ Capability 调用序列（Exec）→ 终态。
3. **定位问题**：若 status=failed，查 Exec Log 中 failure_mode、error、capability_id；若 Gate Reject，查 Trace Log 中 gate_result、reason。
4. **复现**：在非生产环境用相同 entry_type、payload（或从 Snapshot 还原）重放；不修改历史 Execution 数据。

---

## 三、约束

- **只读**：回放与调试不写入生产 task_executions / audit_logs；复现可在沙箱或副本进行。
- **权限**：生产 Audit 访问按 FSD 02.3 Layer Audit And Ownership 与安全策略控制。
- **与 03.4 一致**：所有 Log 含 execution_id（或可关联）；支持按 execution_id 查询完整链路。
